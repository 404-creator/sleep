# 睡觉监督局

情侣早睡监督小应用：设置睡觉和起床时间，到点提醒，超时调侃，早睡打卡攒积分，兑换奖励，本地备份和导入。

## 功能

- 设置睡觉时间、起床时间、宽限时间、提醒频率
- 到点浏览器通知，超时后按间隔继续提醒
- 严格监督模式：超时后提醒更频繁
- 睡前收手机提醒：临睡前单独提醒减少屏幕刺激
- 三种调侃强度：温柔、普通、嘴硬但关心
- 早睡打卡、连续天数、本周按时率、积分
- 今日行动清单：按当前时间给出下一步建议
- 行动建议池：按“未到点、睡前、宽限、超时、已打卡”自动抽取不同建议
- 今晚请假：特殊情况可以记录休息日，不当作熬夜补救
- 趋势页：查看近 14 天按时率、熬夜次数、请假次数和睡眠质量
- 计划页：设置每周目标、明早期待状态、晚间约定清单
- 醒后复盘：记录睡眠质量、精神状态和复盘备注
- 7 日早睡挑战：显示连续进度和挑战提示
- 情侣奖励清单：内置 26 个奖励，支持分类筛选、新增、兑换、删除
- 随机奖励：从当前积分可兑换的奖励里抽一个
- 睡前收尾清单和呼吸放松
- 睡眠监督报告：近 7 次打卡、计划睡眠时长、醒后精神趋势、页面活跃状态
- 本地 JSON 备份和导入
- PWA 配置，支持添加到手机桌面

## 本地预览

推荐用本地服务打开，这样浏览器通知和 PWA 行为更接近线上。

```powershell
.\start.ps1
```

或者：

```powershell
python -m http.server 5173
```

然后打开：

```text
http://127.0.0.1:5173
```

## 用 GitHub Actions 部署到 GitHub Pages

1. 在 GitHub 新建仓库，比如 `sleep-bureau`。
2. 把本目录所有文件上传到仓库根目录，包括 `.github/workflows/deploy-pages.yml`。
3. 进入仓库 `Settings`。
4. 左侧打开 `Pages`。
5. `Build and deployment` 里的 `Source` 选择 `GitHub Actions`。
6. 回到仓库首页，把文件提交到 `main` 分支。
7. 打开 `Actions`，等待 `Deploy GitHub Pages` 工作流完成。
8. 部署成功后，访问链接一般是：

```text
https://你的GitHub用户名.github.io/sleep-bureau/
```

如果你的用户名是 `404-creator`，仓库名是 `sleep-bureau`，链接就是：

```text
https://404-creator.github.io/sleep-bureau/
```

## 手机上使用

iPhone：

1. 用 Safari 打开 GitHub Pages 链接。
2. 点击底部“分享”按钮。
3. 选择“添加到主屏幕”。
4. 点“添加”。

Android：

1. 用 Chrome 打开 GitHub Pages 链接。
2. 点击右上角三个点。
3. 选择“添加到主屏幕”或“安装应用”。
4. 确认添加。

## 注意

数据只保存在当前浏览器本地。清理浏览器数据、换手机、换电脑前，请先在“备份”页面下载备份文件。

浏览器无法判断人是否真的睡着，也不能监控整台电脑；当前监督依据是设置时间、打卡记录、醒后复盘和本页面活跃状态。

如果你已经部署过旧版本，手机上仍看到旧界面，可以刷新一次，或者等待 service worker 更新到新缓存版本。
